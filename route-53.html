AWS Route 53: The Complete Guide for SAA-C03

Everything you need to know about Amazon Route 53 — DNS fundamentals, hosted zones, record types, Alias vs. CNAME, all routing policies, health checks, failover, domain registration, Route 53 Resolver, DNSSEC, availability, cost, hard limits, and every exam trap — in one place.

What is Route 53?

Amazon Route 53 is a highly available, scalable DNS web service that performs three functions:

  1. Domain registration
  2. DNS routing
  3. Health checking

Why "Route 53"? It is named after TCP/UDP port 53, the standard DNS port.

Key properties:

  • 100% availability SLA
  • Global service
  • Anycast network for routing queries to the nearest edge location

DNS fundamentals - what you need to know

How DNS works

  1. User types www.example.com into a browser.
  2. The browser asks a DNS resolver.
  3. The resolver queries root servers, then TLD servers, then the authoritative name servers.
  4. Route 53 returns the IP address.
  5. The browser connects to that IP.

Before a migration or failover, lower TTL well in advance. After the change is stable, raise it again.

Alias records have TTL set automatically by Route 53.

Routing policies - the complete guide

1. Simple routing

PropertyDetail
How it worksReturns one or more values; the client chooses
Health checksNot supported
Use caseSingle resource with no special routing need

2. Weighted routing

PropertyDetail
How it worksDistributes traffic according to weights from 0 to 255
Health checksSupported
Use caseA/B tests, blue-green rollout, gradual migration

Weight 70 to A and 30 to B sends roughly 70% to A and 30% to B. If all records have weight 0, Route 53 distributes traffic equally.

Health checks

Three types of health checks

TypeWhat it monitors
Endpoint health checkHTTP, HTTPS, or TCP check against an IP or domain
Calculated health checkCombines other health checks using threshold logic
CloudWatch alarm health checkTracks a CloudWatch alarm state

Endpoint health check details

  • Route 53 health checkers run from multiple Regions.
  • HTTP and HTTPS checks need a 2xx or 3xx response.
  • You can optionally match a string in the first 5,120 bytes of the body.
  • Intervals are 30 seconds or 10 seconds.
  • Failure threshold defaults to 3 consecutive failures.

Health checks and private resources

Public Route 53 health checkers cannot reach private resources in private subnets. For private targets, publish a metric, create a CloudWatch alarm, and make Route 53 watch the alarm.

Domain registration

  • Route 53 can register new domains and transfer existing ones.
  • Registering a domain automatically creates a matching public hosted zone.
  • DNS hosting and domain registration are separate functions.
  • Domain locking helps prevent unauthorized transfer.
  • WHOIS privacy helps protect contact details.

Route 53 Resolver - hybrid DNS

Default VPC DNS

Every VPC has a built-in resolver at the base address plus 2, such as 10.0.0.2.

Resolver endpoints

EndpointDirectionPurpose
Inbound endpointOn-premises to AWSLets on-prem resolvers resolve AWS private hosted zones
Outbound endpointAWS to on-premisesLets AWS resolve on-prem DNS names

Resolver rules

  • Conditional forwarding rules send selected domains to chosen resolvers.
  • System rules override forwarding for specific domains.
  • Rules can be shared across accounts with AWS RAM.

Route 53 Traffic Flow

Traffic Flow is a visual editor for building and versioning complex routing policies. It is required for geoproximity routing and carries a monthly charge per policy record.

DNSSEC

Route 53 supports DNSSEC signing for public hosted zones only. It cryptographically signs DNS responses to protect against spoofing and tampering.

  • Private hosted zones do not support DNSSEC.
  • KMS keys for DNSSEC must be in us-east-1.
  • DNSSEC increases response size.

Common exam traps and edge cases

  1. CNAME cannot be at zone apex.
  2. Alias queries to AWS resources are free; CNAME queries are not.
  3. Simple routing does not support health checks.
  4. Geolocation is not the same as latency routing.
  5. Failover needs a health check on the primary.
  6. DNS failover is not instant because TTL matters.
  7. Route 53 health checkers cannot reach private resources.
  8. Alias cannot point directly to an EC2 public IP.
  9. Geolocation needs a default record.
  10. Multivalue answer is not a true load balancer.
  11. Private hosted zones need VPC DNS settings enabled.
  12. Domain registration and DNS hosting are separate services.

Final mental model

Every Route 53 exam question can be broken down with this sequence:

  1. What type of record? Apex usually means Alias. Non-AWS domain targets often mean CNAME. IP addresses mean A or AAAA.
  2. What routing is needed? Match the policy to the scenario.
  3. Is health checking needed? Private targets require CloudWatch alarm health checks.
  4. Public or private DNS? Public hosted zone for internet-facing names, private hosted zone for VPC-only resolution.
  5. Is hybrid DNS needed? Resolver inbound and outbound endpoints solve that.
  6. Is failover speed critical? DNS TTL is the limiter; Global Accelerator is the fast-failover answer.
  7. What about cost? Alias is free for AWS resources and TTL affects query volume.

If you can answer these seven questions for any scenario, you can answer any Route 53 question on the SAA-C03.
